「見た目」から「予約と応募が増える」

現行サンプルの内容（コンセプト・料金・アクセス等）はそのまま活かしつつ、 白基調のデザイン刷新・スマホ最適化・主要導線への即アクセス・求人フォーム新設を柱に、 集客と採用の2つの導線を強化するための計画書です。

目的集客導線 × 求人導線の強化
対象fuss.html（1ページ構成）
使用技術HTML / CSS / JavaScript(jQuery)
作成日2026-07-04
01 — OBJECTIVE

このリニューアルで達成したいこと

「情報が載っているサイト」から「行動してもらえるサイト」へ。ゴールは大きく2つです。

1集客導線の強化

訪問者を迷わせず、予約・電話・アクセス・料金という「今すぐ知りたい／今すぐ動きたい」情報へ1タップで到達させ、予約ページへの遷移率を高めます。

2求人導線の強化

現状はゼロの採用の受け皿を新設。募集要項＋かんたん応募フォームを用意し、「気になった人がその場で応募できる」状態をつくります。

02 — AS-IS

現状サンプルの課題

内容は良いのに「伝わり方」と「動線」で機会を損失している状態です。

!
主要情報が縦一列で埋もれている予約・料金・TEL・アクセスに辿り着くまでスクロールが必要。スマホでは特に離脱の原因に。
!
常時表示のナビ・予約ボタンがないページ下部まで読んで初めて予約導線に出会う構造。「今すぐ予約」の取りこぼし。
!
求人の受け皿がゼロ採用ページも応募窓口も存在せず、人材の問い合わせを一切拾えていない。
!
視覚的な魅力・世界観が弱い写真・スタイル事例・スタッフの顔が乏しく、来店イメージが湧きにくい。
!
スマホ最適化が不十分文字サイズ・余白・タップ領域がPC寄り。多数を占めるスマホ来訪の体験が最適でない。
!
SEO/MEO・SNS連携が未整備meta情報・構造化データ・Instagram等の連携がなく、検索と回遊で不利。
03 — DESIGN

デザイン方針：白基調 × レスポンシブ

清潔感と上質さを伝えるホワイトベースに、木目を思わせるアクセント1色。PC・スマホどちらでも崩れない設計にします。

🤍 白基調・余白設計

背景は白（#fff）を主役に、罫線とごく淡いグレーで情報を整理。差し色はゴールド／木目系1色に絞り、上品で写真映えする画面に。

📱 モバイルファースト

スマホ幅を基準に設計し、PCは広がるだけ。タップ領域44px以上、読みやすい文字サイズと行間、片手で完結する操作性。

⚡ 軽さ・表示速度

画像はwebpで軽量化、遅延読み込み（lazy）。外部ライブラリはjQuery程度に留め、素早く開くページに。

この計画書自体が方針の見本です。 白基調・上部固定ナビ・スマホでのレイアウト折返しなど、提案する要素を実際に体感いただけるよう作っています。

挙動の仕様

  • 予約：楽天ビューティの予約ページを新規タブで開く（外部遷移）
  • 料金・アクセス：ページ内の該当セクションへスムーズスクロール
  • TEL：tel:リンクでスマホは即発信、PCは番号を表示

実装イメージ

  • スマホ：position:fixed; bottom:0 の4分割タブバー
  • PC：position:sticky; top:0 のヘッダーナビ
  • スクロール連動（jQuery）で現在地セクションをハイライト
  • 「予約」だけ色を変え最も押されやすく（プライマリ扱い）
05 — KEY FEATURE

求人・応募フォーム MUST

採用セクション（募集要項）＋シンプルな応募フォームを新設。送信するとお店のメールへ届く、複雑さのない構成です。別ページ（recruit.html）としても、同一ページ内セクションとしても実装可能です。

応募フォーム（レイアウト見本）

応募する（送信）

※ 見本のため入力は無効化しています。

送信方式の選択肢

  • 推奨：フォームサービス連携（Formspree / Googleフォーム / FormRun 等）— サーバー不要で店舗メールへ届く。迷惑メール対策・自動返信も可
  • mailto方式— 最も簡易。訪問者のメーラーが起動（環境依存があり補助的）
  • PHP等が使える場合— 自前の送信スクリプトで完全自由（要サーバー）

備える要素

  • 募集要項（職種・給与・勤務時間・歓迎スキル・お店の魅力）
  • 必須項目の入力チェック（jQueryでバリデーション）
  • 送信完了メッセージ／サンクス表示
  • 個人情報の取り扱いに関する一文＋同意チェック
  • スパム対策（簡易なbot除け／reCAPTCHA等）
06 — RECOMMENDATIONS

集客・求人を伸ばす追加提案 IDEA

マスト要件に加え、費用対効果の高い施策を優先度つきでご提案します。実施は選択いただけます。

集客を伸ばす

📸 スタイルギャラリー

ビフォーアフター／得意スタイルの写真集。Instagram連携で更新の手間なく“作品”を見せ、指名・来店動機を作る。

🙋 スタッフ紹介

顔・人柄・得意分野を掲載。「この人にお願いしたい」を生み、マンツーマンの強みを可視化。

⭐ お客様の声・口コミ

楽天ビューティ／Googleの口コミを掲載し信頼を担保。星評価は来店の後押しに直結。

🎟 初回クーポン・特典

「初回カット10%OFF」等を目立たせ、新規のハードルを下げる。予約導線とセットで訴求。

💬 LINE公式連携

予約・再来店・キャンペーン告知の受け皿。友だち追加ボタンでリピート導線を確保。

❓ よくある質問

駐車場・支払い方法・子連れ可否など。不安を先回りで解消し予約前の離脱を防ぐ。

検索・回遊で見つけてもらう（SEO / MEO）

🔎 meta / OGP整備

title・description・OGP画像を設定し、検索結果とSNSシェア時の見え方を最適化。

📍 MEO / 構造化データ

Googleビジネスプロフィール連携とLocalBusiness構造化データで「新宿御苑前 美容室」の地図検索に強く。

📈 アクセス解析

GA4等を設置し、どこから来てどこで離脱するかを可視化。改善を数字で回す土台に。

採用を伸ばす

💼 働く環境の見せ方

給与例・待遇・1日の流れ・スタッフの声を掲載。「応募したくなる理由」を具体化。

🔗 応募のしやすさ

フォーム＋LINE応募＋電話の複数導線。求人媒体からの流入も受け止める。

🖼 サロンの雰囲気

店内・チームの写真で職場イメージを伝え、ミスマッチと不安を軽減。

07 — STRUCTURE

サイト構成案（1ページ + 求人ページ）

基本は縦1ページのランディング型を踏襲。求人だけ独立ページに分け、上部/下部ナビでどこからでも主要情報へ。

#セクション役割・ねらい導線
1固定ナビ / 予約タブいつでも主要導線へ（集客の要）予約・料金・アクセス・TEL
2ヒーロー（メインビジュアル）第一印象・キャッチ・予約ボタン→ 予約
3当店の特徴マンツーマン等の強みを訴求—
4スタイルギャラリー IDEA作品で来店イメージ醸成→ Instagram
5料金明朗会計で安心（頻繁に見られる）→ 予約
6スタッフ紹介 IDEA指名・信頼の獲得—
7お客様の声 IDEA口コミで背中を押す—
8アクセス / 地図来店前に必ず見る情報→ Map / TEL
9店舗情報・FAQ営業時間・不安解消—
10予約・電話最終CTA→ 予約 / TEL
11採用情報 → 応募フォーム MUST求人導線（別ページ recruit.html）→ 応募送信

09 — PROCESS

STEP 5公開・改善
